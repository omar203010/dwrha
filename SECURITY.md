# سياسة الأمان

## المعلومات الحساسة

⚠️ **مهم جداً**: لا ترفع الملفات التالية على GitHub أو أي مستودع عام:

- `.env` - يحتوي على `SECRET_KEY` ومعلومات قاعدة البيانات
- `db.sqlite3` - قاعدة البيانات المحلية
- `*.log` - ملفات السجلات
- أي ملف يحتوي على كلمات مرور أو مفاتيح API

## كيفية الحفاظ على الأمان

### 1. استخدام `.env` للمعلومات الحساسة

جميع المعلومات الحساسة يجب أن تكون في ملف `.env` الذي:
- ✅ موجود في `.gitignore`
- ❌ لا يتم رفعه على GitHub

### 2. إنشاء `.env` من القالب

```bash
cp .env.example .env
```

ثم عدل `.env` وأضف:
- `SECRET_KEY` - مفتاح Django السري (يُنشأ بشكل عشوائي)
- معلومات قاعدة البيانات (في الإنتاج)
- معلومات البريد الإلكتروني (إن وُجد)

### 3. إنشاء SECRET_KEY آمن

```bash
python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'
```

### 4. إعدادات الأمان في الإنتاج

- `DEBUG=False` - تعطيل وضع التطوير
- `SECURE_SSL_REDIRECT=True` - إجبار HTTPS
- `SESSION_COOKIE_SECURE=True` - حماية الكوكيز
- `CSRF_COOKIE_SECURE=True` - حماية CSRF
